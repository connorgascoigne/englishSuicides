## üîß Running the Code

In general, all code-related files are designed to be as automated as possible. However, a few manual steps are required to get everything working after cloning the repository.

### 1. üì¶ Installing Required Packages

Each `.R` file includes a section titled `## 0.1. packages`. This section will attempt to load or install the necessary `R` packages.

üß† Note: One package, `R-INLA`, is not on CRAN and requires a custom installation.

Use the following code to install R-INLA:

```{r}
if (!require('INLA')) {
  install.packages(
    'INLA',
    repos = c(getOption('repos'), INLA = 'https://inla.r-inla-download.org/R/testing'),
    dependencies = TRUE
  )
}
```

If installation issues occur, please consult the [R-INLA Download & Install guide](https://www.r-inla.org/download-install).

### 2. üìÅ Fixing the Working Directory

Each `.R` script contains a section titled `## 0.2. directories`. The working directory is determined using:

```{r}
dir.path <- rstudioapi::getActiveDocumentContext()$path
```

This works only in RStudio (and only if the file is saved). Then, the root directory is extracted using:

```{r}
dir.home <- sub('(englishSuicides).*', '\\1', dir.path)
```

üìù Action Required: If you've renamed the englishSuicides folder, update "englishSuicides" in the line above to match your local folder name.

### 3. üìÇ Providing the Master Data set

Within the `## 0.2. directories` section in each `.R` file, another directory is referenced to access the required external source data (e.g., mid-year population, IMD, etc.). This folder is not included in the repository.

This is defined as:

```{r}
dir.masterData <- paste0(sub('(OneDrive - Imperial College London).*', '\\1', dir.path), '/00_masterData')
```

üì• Action Required: Download the necessary data (see [üóÇÔ∏è Data Sources] below) and store it in the above folder path on your local machine.

##### üìÅ `00_simulatingData/`

As we are unable to share the data, the files here simulate the suicide, population and covariate data. Once these data have been simulated and saved, run only üìÑ `01_sortSpatialData.R` from the üìÅ `01_processingCovariateData/` folder, and then proceed to üìÅ `02_processingSuicideData/`.

üìù Action Required: If you're simulating data, then make sure you have the right files being called in all files inside the folders üìÅ `02_processingSuicideData/` and üìÅ `03_mainResults/`.

#### üìÅ `01_processingCovariateData/`

In this folder, we are orgnaising all the non-suicide related data. The sources for all the data can be found in the table in [üóÇÔ∏è Data Sources]. Only the file üìÑ `01_sortSpatialData.R`, which organises the spatial data for the modelling, needs to be run if the simulated data is being used. The naming convection of the files describes what data is being sorted. Remember, make sure these files are run in order due to dependencies.

üó∫ Additional spatial levels: The spatial resolution of the analysis was the Middle layer Super Output Area (MSOA). However, when organising the covariates (files located in üìÅ `code/01_processingCovariateData`), we have included code that produced each at the Lower layer Super Output Area (LSOA) and Local Authority District (LAD) area as well. For the analysis of suicides in England from 2002 to 2022, these parts of the code can be ignored.

#### üìÅ `02_processingSuicideData/`

The suicide data from the ONS (and the simulated suicide data) comes aggregated by year, month, LSOA, age, and sex.

The file üìÑ `01_sortSuicideData.R` takes the extract suicide data and i) aggregates into year, MSOA, age and sex, ii) performs the age-sex standardisation to format the data into year and MSOA only, and iii) joins the covariates data onto it. Once the covariates are joined to the suicide data, they can be explored to understand the format.

The file üìÑ `02_exploreCovariateData.R` produces three main outputs i) histogram for each of the covariates, ii) correlation between each of the covariates and iii) correlation between the suicide counts, the IMD and the domains that make up the IMD.

##### üìÅ `01_covariateData`

This folder stores the figures that explore the covariates.

The üìÅ `gif/` folder contains `.gif` files which are animations of the covariates that vary in time. This is to see the spatial and temporal dynamics.

The üìÅ `png/` folder contains `.png` files of the the covariates averaged over time. This is to see the spatial pattern, but not the temporal one.

The üìÅ `txt/` folder contains `.txt` files which list the spatial areas that needed to be imputed.

For all the folders, there are subs folders for `üìÅ lsoa/` and `üìÅ msoa/`. This is due to the code being set up to create outputs at LSOA, MSOA and LAD levels. Furthermore, some covariates (IMD, ethnicity population totals, and population totals) are downloaded at the LSOA level and then are aggregated to the MSOA level. Missing data is imputed at the LSOA level.

##### üìÅ `02_suicideData/`

This folder stores the figures that explore the covariates data (i.e., histograms and correlations) once joined to the suicide data.

##### üìÅ `03_modelFit/`

This folder stores the results of fitting the model and extracting the posterior distribution. As this may contain elements of the true data, we are unable to share these files due to data governance.

##### üìÅ `04_mainResults/`

This folder stores the results of the main analysis that are presented in the paper and the supplementary material.
